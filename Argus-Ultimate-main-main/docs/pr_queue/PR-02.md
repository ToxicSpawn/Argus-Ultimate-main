# PR-02 Order Intents Idempotency

## Scope
- Add `order_intents` lifecycle storage (`CREATED -> SENT -> ACKED -> FILLED/CANCELED/FAILED`).
- Use deterministic `intent_id` generation for retries and reconnects.
- Enforce resend guards so retries do not duplicate exposure.
- Link exchange order ids back to the intent record.
