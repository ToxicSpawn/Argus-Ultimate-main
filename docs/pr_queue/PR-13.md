# PR-13 Reconciliation Ownership + Operator Ack

## Scope
- Run reconciliation every live cycle (paper optional toggle).
- Freeze all new intents when drift exceeds threshold.
- Require explicit operator acknowledgment to unfreeze after halt.
- Log/alert full drift diff and remediation actions.

## Primary Files
- `unified_trading_system.py`
- `unified_execution_engine.py`
- `execution/reason_codes.py`
- `monitoring/alerting.py`

## Verification
- `py -3 -m unittest tests_unified.test_pr_11_to_20_hardening.TestPR13ReconOwnership -v`

## Operator Recovery
- `python main.py reconcile-ack --config unified_config.yaml --operator-id oncall_1`
- Command requires exact `YES` confirmation (or `--yes-ack` in non-interactive automation).

## Audit Query Examples
- `SELECT event_type, reason_code, freeze_id, datetime(timestamp, 'unixepoch') AS ts FROM reconciliation_events ORDER BY timestamp DESC LIMIT 20;`
- `SELECT trace_id, reason_code, details_json FROM decision_snapshots WHERE reason_code IN ('RECONCILIATION_HALT','OPERATOR_ACK_REQUIRED') ORDER BY timestamp DESC LIMIT 20;`
